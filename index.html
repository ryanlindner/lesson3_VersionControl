Lesson 3 Highlights

Ryan Lindner

Highlight #1 Version Control

Version Control is the concept of keeping track of all "versions" and revisions of a computer file. This could be any type of file, maybe the changes to a website graphic, a websites css code, or every single piece of code associated with a website. Version control seeks to make tracking a files history and development in an organized way. If a user needs to go back to a previous version of a file, it should be easy to track down and locate the desired file. It's like organizing all your files in a digital file cabinet.

Highlight #2 Git

Git is an example of a version control system. It is a system designed to make accessing and tracking all files in a project easy and well organized. Git can be very useful when a team of web developers are working on a project together. It allows every developer on the team to have a copy of the code on their computer every developer is able to make changes to the code and upload the changes to the remote repository. Every change that each developer makes can be neatly organized by date and other team members can see who made what changes to the code and why they made such changes.

Highlight #3 Local Repository

When using Git, a user will have all of the project files located on their computer inside of what's called a "local repository." The idea is for all team members to have the project files available inside of their own computer locally so that they can edit or make changes to the files without being connected to a server or without even needing internet. If you have 10 team members that all have the project files installed locally on their computer this also serves as a good way to 'back up' all the project files if one of the team members computers dies or gets corrupted.

Highlight #4 Remote Repository

After a user makes changes to the project files locally on their computer, they will then 'push' the changes to what's known as a "remote repository." This is a central storage space on a server that every one of the team members can send their changes and revisions to the code to. This is useful because it acts as the hub between all of the team members working on the project. If a team member needs to re-download all of the files associated with a project, they can do that from the remote repository.

Highlight #5 Basic Git Commands: Add, Commit, Push

Three basic commands used in Git are Add, Commit and Push. A user will use "Add" a file to their local repository. This is needed in order to let Git know that you would like to start tracking the files versioning and changes from here on out. A user will use the "Commit" command when they have finished editing a file from their local repository. It's much like "saving" your changes to a file locally. Once the user is ready to push the newly edited file or files up to the remote repository, they will use the "Push" command. This will update the files on the remote reposity to reflect the changes.
